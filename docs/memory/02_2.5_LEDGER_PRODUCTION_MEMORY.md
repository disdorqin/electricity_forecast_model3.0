
## 核心思想

2.5 已经从 validation tap 转向：

```text
ledger production
```

它不再每天构造 D-30 ~ D-1 validation tap，而是：

```text
每天只做真实预测
预测结果写入 prediction ledger
从第31天开始，用过去30天真实预测 + 实际值学习权重
再融合当天预测
最后做 classifier / correction / final outputs
```

## 典型流程

```text
ledger_predict

```

## 3.0 主链路应仿照 2.5

建议 3.0 主流程：

```text
Stage 0: 数据同步与标准化
Stage 1: 多模型基础预测
Stage 2: prediction ledger 写入
Stage 3: actual ledger 写入 / 对齐
Stage 4: 过去30天 ledger 权重学习
Stage 5: risk / regime / assist signals 生成
Stage 6: regime-aware fusion
Stage 7: negative / residual plugin 修正
Stage 8: guardrail fallback
Stage 9: final outputs + delivery report
```

## 3.0 不应回退

- 不回退到 validation tap 作为生产主线。
- 不为了学习权重而模拟历史预测。
- 不把模型输出自带 y_true 当 truth。
- 不用自然日切分。
